# Twiv wiki/canon 스키마

`characters/<slug>/canon.md` 와 Notion `캐릭터 도메인/<이름>/wiki` 는 **이 계약을 1:1로 따른다.**
여기는 사람이 읽는 정본이다. 실제 플랫폼 복붙본은 `profile.twiv.md`, 장기 운용 규칙은 `lorebook.md`로 분리한다.

> **Twiv은 영상 우선(릴스/쇼츠형) 플랫폼이다.** liveher(인스타형 사진 피드)와 달리, 캐릭터의 첫 표면은
> **세로 온보딩 영상**이다. 사진은 보조 갤러리다. 그래서 canon에는 온보딩 영상이 전달할 훅/첫 대사를 분명히 둔다.

## 필수 필드

| 필드 | 뜻 | 길이 가이드 |
| --- | --- | --- |
| 🪪 캐릭터 이름 | 실제 캐릭터명. 컨셉 타이틀과 분리 가능 | 1개 |
| 📺 채널 계정 | 릴스형 피드/프로필에서 쓰는 표시명과 핸들 | 표시명 1개 + handle 1개 |
| 🪝 캐릭터 소개 | 첫 화면(온보딩 영상 위) 훅 / 한 줄 소개 | 1~3문장 |
| 📖 스토리 개요 | 설정·배경·관계 훅. 이름 없는 상대와의 포지션을 분명히. | 3~6문장 |
| ✨ 관심사 | 캐릭터가 진심인 주제 | 2~4개 |
| 🎯 취미 | 실제로 하는 것 | 2~4개 |
| 💬 좋아하는 말 | 캐릭터가 듣고 좋아하거나, 관계 압력을 여는 말 | 1~3줄 |
| 💕 이상형 | 어떤 상대에게, 어떤 상황에서 끌리는지 | 1~3문장 |
| 👋 첫 인사말 | 온보딩 영상 직후 대화 시작 첫 발화 | 1줄 |

## 보조 필드

- 🎬 **온보딩 영상 훅** — 세로 온보딩 영상이 6초 안에 전달할 인상(표정/동작/분위기) 메모. 실제 파일은 `media/온보딩.mp4`.
- #️⃣ **해시태그** — meta `hashtags` 와 동기화

## 규칙

- 성인 현재시점 고정. 미성년/고등학생 코드 금지. (자세한 감리 라인은 `copy_rules.md`)
- 남자 주인공/사용자 이름을 고정하지 않는다. 노출문은 `너`, `상대`, `그 사람`, 관계 호칭(예: `선배`)처럼 이름 없는 관계로 처리한다.
- `canon.md` = wiki 정본. 폼에 붙여넣을 다듬은 버전은 `profile.twiv.md` 로 **투영**(ACL 단방향).
- 로어/금지선/호감도/아이템 해석은 `lorebook.md`로 보낸다.
- 영상/이미지는 여기 적지 않는다 → `media/INDEX.md`, `images/INDEX.md`.
